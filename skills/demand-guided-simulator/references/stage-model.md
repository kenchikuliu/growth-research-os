# Stage Model

This skill follows the `web.cafe` simulator pattern in product shape, not by copying page text.

## Required Product Moves

1. Start from a contradiction.
   Example:
   - `这个词看起来能做，但它到底有没有被搜索稳定承接？`
   - `这个站看起来在涨，但到底是哪类页面在涨？`

2. Name the hidden variable.
   In this repo, common hidden variables are:
   - page type
   - search carry
   - landing-page proof
   - clusterability
   - chain closure

3. Offer two entry paths.
   - `跟我走完分阶段诊断`
   - `直接看最终建议`

4. Use bounded steps.
   - `8` steps for demand validation
   - `6` steps for attribution

5. Keep one concept per step.
   Do not mix trend interpretation, page type, monetization, and execution all in one panel.

## Demand Flow

1. Frame the demand correctly
2. Check whether demand is real
3. Check whether search is carrying it
4. Identify winning page shape
5. Check whether it can expand into a cluster
6. Read trend durability
7. Judge monetization and execution fit
8. Commit to stop / watch / one page / cluster / site

## Attribution Flow

1. Freeze the time window
2. Identify which pages actually moved
3. Identify which terms and page types drove movement
4. Check whether the pattern repeats structurally
5. Separate reusable from non-reusable factors
6. Close the attribution chain and turn it into an action

## Evidence Rule

Each step must separate:

- facts from tools
- inference from rules
- diagnosis
- next action

Do not jump directly from score band to recommendation without showing why the band is earned.
